# 05-部署手册

## 方案 A：Render 部署（推荐，完全免费）

### 步骤 1：准备代码
确保你的代码已提交到 GitHub 仓库。

### 步骤 2：注册 Render
1. 访问 https://render.com
2. 使用 GitHub 账号登录
3. 授权 Render 访问你的仓库

### 步骤 3：创建 Web Service
1. 点击 Dashboard → "New +" → "Web Service"
2. 选择你的 `bonnie-fridge` 仓库
3. 配置如下：

| 配置项 | 值 |
|--------|-----|
| Name | `bonnie-fridge-api` |
| Environment | `Python 3` |
| Build Command | `pip install -r backend/requirements.txt` |
| Start Command | `cd backend && uvicorn main:app --host 0.0.0.0 --port $PORT` |
| Plan | Free |

4. 点击 "Advanced" → 添加环境变量：
   - `KIMI_API_KEY` = `sk-your-kimi-key`
   - `DAILY_LIMIT` = `10`

5. 点击 "Create Web Service"

### 步骤 4：等待部署
- Render 自动构建并部署
- 约 2-3 分钟后，你会获得一个类似 `https://bonnie-fridge-api.onrender.com` 的地址
- 访问 `https://bonnie-fridge-api.onrender.com/docs` 查看 API 文档

### 步骤 5：更新前端 API 地址
打开 `frontend/js/app.js`，修改：
```javascript
const API_BASE = "https://bonnie-fridge-api.onrender.com";  // 你的 Render 地址
```

### 步骤 6：部署前端（GitHub Pages）
1. 进入仓库 Settings → Pages
2. Source 选择 "Deploy from a branch"
3. Branch 选择 `main`，文件夹选择 `/frontend`
4. 保存后获得 `https://yourname.github.io/bonnie-fridge/frontend/`

## 方案 B：本地开发（调试使用）

### 后端启动
```bash
cd backend
python -m venv venv
source venv/bin/activate  # Windows: venv\Scripts\activate
pip install -r requirements.txt
# 创建 .env 文件
echo "KIMI_API_KEY=sk-your-key" > .env
echo "DAILY_LIMIT=10" >> .env
uvicorn main:app --reload
```

### 前端访问
直接双击打开 `frontend/index.html`，或使用 Live Server 插件。

## 常见问题

**Q: Render Free 服务会休眠？**
A: 是的，15 分钟无访问会休眠，下次访问需等待 30 秒唤醒。面试前先发个请求"暖机"。

**Q: 如何更新代码？**
A: 推送到 GitHub 后，Render 自动重新部署。

**Q: 前端和后端域名不同，跨域怎么办？**
A: 后端已配置 `CORSMiddleware(allow_origins=["*"])`，无需额外处理。
