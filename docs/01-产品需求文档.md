# 01-产品需求文档（PRD）

## 1. 项目概述

| 项目属性 | 说明 |
|---------|------|
| 项目名称 | Fridge2Fork · 冰箱侦探 |
| 一句话介绍 | 拍一张冰箱照片，AI 识别食材并推荐"现在就能做的菜" |
| 目标用户 | 日常做饭的普通人，尤其是不知吃什么的上班族 |
| 核心价值 | 把"看冰箱想菜单"的烦人事交给 AI 秒级解决 |
| 开源协议 | MIT |

## 2. 用户痛点
- 打开冰箱不知道能做什么菜
- 食材放到过期才想起
- 做饭时才发现缺关键调料
- 不知道如何处理剩余食材

## 3. 功能模块

### 3.1 AI 食材识别（多模态入口）
- 用户拍摄/上传冰箱内部照片
- AI 识别食材名称、大致数量
- **规则**：识别到类别即可（如"酱油"），不识别品牌
- **规则**：不支持手写标签识别
- 支持语音/文字补充食材（如"冷冻层还有牛排"）
- 对每项识别结果标注置信度（高/中/低）

### 3.2 用户画像
- 口味偏好：清淡 / 微辣 / 重辣 / 酸甜 / 不限制
- 饮食目标：减脂 / 增肌 / 控糖 / 素食 / 无特殊
- 烹饪时长：15分钟内 / 30分钟内 / 不限制
- 常备食材：用户勾选默认有的调料

### 3.3 智能菜谱推荐（双入口）

#### 入口 A：被动推荐（核心）
- 基于库存推荐 3-5 道菜
- 排序逻辑：匹配度 + 新鲜度优先（优先消耗易过期食材）
- 每道菜显示：已有食材 ✓ / 缺料 ✗ / 推荐理由

#### 入口 B：反向查询（加分）
- 用户输入"想做红烧肉"
- AI 检查库存 → 反馈"已有/缺料"清单

### 3.4 缺料购物清单
- 每道推荐菜可一键加入购物清单
- 自动去重、标注用途
- 纯文本导出（复制到剪贴板）

### 3.5 库存管理
- 手动纠偏：删除误识别、添加漏识别
- 消耗跟踪：确认做菜后自动扣减库存
- 常备食材：作为默认存在，不依赖拍照

## 4. 非功能需求

| 维度 | 要求 |
|------|------|
| 性能 | 识别响应 < 5 秒，推荐响应 < 1 秒 |
| 成本 | 每日调用上限 10 次（演示 Key） |
| 安全 | API Key 仅存储于后端环境变量，不暴露前端 |
| 兼容性 | 支持 iOS Safari / Android Chrome |
| 离线 | MVP 不要求离线功能 |

## 5. 面试展示要点
1. **多模态 AI 应用**：真实调用视觉大模型
2. **完整产品闭环**：识别 → 推荐 → 清单 → 消耗跟踪
3. **双入口设计**：被动推荐 + 主动搜索
4. **成本意识**：限流 + BYOK 混合架构
