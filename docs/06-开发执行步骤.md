# 06-开发执行步骤

## Phase 1：环境搭建（第 1 天）

- [ ] 安装 Python 3.10+
- [ ] 安装 VS Code（推荐）
- [ ] 创建 GitHub 仓库
- [ ] 克隆仓库到本地
- [ ] 创建虚拟环境并安装依赖
- [ ] 申请 Kimi API Key（https://platform.moonshot.cn）
- [ ] 验证后端能启动（`uvicorn main:app --reload`）
- [ ] 验证前端能打开（双击 `index.html`）

**交付物**：本地能跑通 Hello World

---

## Phase 2：核心链路打通（第 2-3 天）

- [ ] 实现 `/api/ingredients/recognize`（拍照 → AI 识别 → 返回 JSON）
- [ ] 前端实现拍照上传 + 展示识别结果
- [ ] 手动纠偏功能（删除/添加食材）
- [ ] 语音补充食材（浏览器 Web Speech API）

**交付物**：拍冰箱 → 看到食材列表（能手动改）

---

## Phase 3：推荐引擎（第 4-5 天）

- [ ] 整理 15+ 道家常菜的 JSON 数据
- [ ] 实现匹配算法（食材覆盖度计算）
- [ ] 实现 `/api/recipes/recommend`
- [ ] 前端展示推荐卡片（已有 ✓ / 缺料 ✗）
- [ ] 用户画像页面（口味/目标/常备食材）
- [ ] 画像持久化到后端内存

**交付物**：拍完照 → 看到 3-5 道推荐菜

---

## Phase 4：反向查询 + 购物清单（第 6 天）

- [ ] 实现 `/api/recipes/check`（想做某菜 → 查缺料）
- [ ] 前端搜索框 + 结果展示
- [ ] 实现 `/api/recipes/shopping-list`
- [ ] 前端购物清单页面（增删改查 + 复制导出）
- [ ] 消耗跟踪（做菜后扣减库存）

**交付物**：完整闭环——拍照/搜索 → 推荐/查缺 → 加购 → 做菜扣库存

---

## Phase 5：Polish + 开源（第 7 天）

- [ ] UI 美化（Tailwind 微调、动画、配色）
- [ ] 录演示视频/GIF
- [ ] 完善 README（产品故事 + 架构图 + 部署教程）
- [ ] 检查 `.gitignore`（确认 `.env` 不在仓库中）
- [ ] 部署到 Render
- [ ] 部署前端到 GitHub Pages
- [ ] 写知乎/即刻文章引流

**交付物**：线上可访问、GitHub 仓库完整、README 专业

---

## 每日检查清单

每天结束前确认：
1. 今天的代码能运行吗？
2. `.env` 文件是否被意外提交？
3. 是否有需要补的注释？
4. 是否写了有意义的 git commit message？

---
