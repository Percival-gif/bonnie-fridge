## 🛠️ 工作流说明

### 开发工作流
```
需求确认 → 阅读对应 docs → 编码 → 自测 → 提交 Git → 更新文档（如有变更）
```

### 文档更新规范
- 如果修改了 API 接口，必须同步更新 [03-API接口规范.md](docs/03-API接口规范.md)
- 如果修改了技术架构，必须同步更新 [02-技术架构.md](docs/02-技术架构.md)
- 如果新增了功能模块，必须同步更新 [01-产品需求文档.md](docs/01-产品需求文档.md)
- 文档与代码不一致时，以代码为准，但文档应在 24 小时内补齐

### 安全红线（绝对禁止）
- ❌ 将 `KIMI_API_KEY` 写入任何 `.py` / `.js` / `.html` 文件
- ❌ 将 `.env` 文件提交到 GitHub
- ❌ 在 README 或文档中展示真实的 API Key
- ❌ 在前端暴露后端服务器的真实密钥
